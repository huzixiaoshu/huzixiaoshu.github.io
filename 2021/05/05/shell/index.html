shell编程

  1. #！ 指定解释器 例如：#!/bin/bash
  2. # 注释
  3. 运行：
  • chmod a+x sample.sh + ./sample.sh
  • /bin/bash sample.sh
  • /bin/sh sample.sh
  • . sample.sh
  • source sample.sh

    内建命令和小括号

  1. man bash-builtins 查看man手册
  2. ( shell 命令 ) 不会改变工作目录
  3. ; 多个命令可用相隔使用

    变量

    按照惯例，shell变量通常由字母加下划线，由任意长度的字母、数字、下划线组成。有两种类型的shell变量
  4. 环境变量
    环境变量可用父进程传给子进程，因此shell 进程的环境变量可用从当前进程传给fork出来的子进程，用printenv命令可以显示当前shell进程的环境变量
  5. 本地变量
    只存在于当前shell进程，用set命令可以显示当前shell进程中定义的所有变量（包括本地变量和环境变量）和函数
  • 可以用export导出，使其为环境变量
  • 用unset命令可以删除已定义的环境变量或本地变量
  • alias取别名

    文件名代换（Globbing）

    这些用于匹配的字符称为通配符（Wildcard），如：* ？ []
  • * 匹配0个或多个任意字符
  • ? 匹配一个任意字符
  • [若干字符] 匹配方括号中任意一个字符的一次出现

    命令代换

  • 转义字符：两种实现方法 ，可以实现原来shell的结果，而不是date字符,显示时间
    • VAR=$(data)
    • VAR=`date`

      算术代换

  1. 比如实现40+33
  • $(($var+33))
  • $((var+33))
  • $[var+33]
  • $[$var+33]
  1. $[base#n]，其中base表示进制，n按照base进展解释，后面再有运算数，按照十进制解释

    转移字符

  2. \反斜杠
  3. 如果创建 类似 –abc这种的文件名，可以使用
  • touch -- --abc
  • touch ./--abc

    单引号

    单引号用于保持引号内所有字符的字面值

    双引号

    被双引号括住的内容，将被视为单一字串。它防止通配符扩展，但允许变量扩展。这点与单引号的处理方式不同

    条件测试

  1. echo $? 返回上一个程序的返回值
  2. 成功返回0,失败了为1
  3. test或者[] 判断
  4. -a逻辑与
  5. -o逻辑或

    作为一种好的shell习惯中，变量取值都放在" "中

分支

  1. if 是模糊匹配
  2. :意思是永远为真
  3. case/esac 是精准匹配
  • esac 表示case语句块的结束
  • 每个匹配分支可以有若干条命令，末尾必须以;;结束，执行时找到第一个匹配的分支并执行相应的命令，然后直接跳出到esca之后。

循环

break 和continue

break[n]可以指定跳出几层循环
continue 跳出本次循环

位置参数和特殊变量

  • $0 代表C中的argv[0]
  • $1 代表C中的argv[1]，后面一次类推
  • $# 相当于argc 表示参数的个数
  • $$ 当前进程号
  • shift 表示左移参数，只是出队参数列表

    重定向

  1. tee显示输出的结果重定向到文件中
  2. cmd > file 把标准输出重定向到新文件中
  3. cmd >> file 追加
  4. cmd > file 2>&1 标准出错也重定向到1所指的file里
  5. cmd < file1 > file2 输入输出都定向到文件里
  6. cmd < &- 关闭标准输入

    脚本测试

  • -n 读一遍脚本程序，不执行，检查错误
  • -v 一边执行一边显示
  • -x 提供追踪执行信息，将执行的每一条命令和结果依次打印出来
    • set -x 和 set +x 分别表示启用和禁用-x参数，这样可以只对脚本中的某一段进行追踪调试。

      正则表达式

      字符类

  • . 匹配任意一个字符
  • [] 匹配括号中的任意一个字符
  • - 在[]内字符的范围
  • ^ 在[]内，匹配括号中的字符之外的任意一个字符
  • [[:xxx:]] grep工具中预定义的一些命名字符类：[[:alpha:]]匹配一个字母，[[:digit:]]匹配一个数字

    数量限定符

  • ?紧跟在他前面的单元应匹配零次或一次
  • +一次或多次
  • *零次或多次
  • {n} 精准匹配n次
  • {n，} 至少匹配n次
  • {，m} 最多匹配n次
  • {n，m} 最少匹配n次，最多匹配m次

    位置限定符

  • ^匹配行首的位置
  • $匹配行尾的位置
  • \<th匹配以th开头的单词
  • \>th匹配以th结尾的单词
  • \b匹配单词开头或结尾的位置
  • \B匹配非单词开头或结尾的词语

    其它限定符

  • \转移字符
  • ()将某个地方框起来组成个单元，使用数量限定符
  • |或
